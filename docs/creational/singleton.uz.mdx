# Singleton patterni

## Sharh

Singleton patterni — bu yaratuvchi (creational) dizayn patterni bo'lib, u klassning instansiya qilinishini faqat bitta obyekt bilan cheklaydi. U ushbu instansiyaga global kirish nuqtasini taqdim etadi.

### Maqsadi

- Ilova butun hayoti davomida klassning faqat bitta instansiyasi mavjudligini ta'minlash
- Ushbu instansiyaga global kirish nuqtasini taqdim etish
- Umumiy resurslarga (ma'lumotlar bazalari, fayl tizimlari, loggerlar va boshqalar) kirishni nazorat qilish

## Muammo

Ba'zan klassning faqat bitta instansiyasi borligiga ishonch hosil qilishingiz kerak bo'ladi. Masalan:
- Ma'lumotlar bazasiga ulanish puli (pool)
- Logger
- Konfiguratsiya menejeri
- Thread pool

Ushbu patternsiz siz tasodifan bir nechta instansiyalarni yaratishingiz mumkin, bu esa mos kelmaydigan holatga (state) yoki resurslarning isrof qilinishiga olib keladi.

## Yechim

Singleton patterni buni quyidagilar orqali hal qiladi:
1. Konstruktorni `private` qilish
2. Klassning `static` instansiyasini yaratish
3. Instansiyaga kirish uchun `static` metodni taqdim etish

## Murakkab: Lazy Initialization (Thread-Safe)

<Tabs defaultValue="typescript">
  <TabsContent value="typescript">
    ```typescript
    class Database {
      private static instance: Database;

      private constructor() {
        console.log('Ma'lumotlar bazasi instansiyasi yaratildi');
      }

      public static getInstance(): Database {
        if (!Database.instance) {
          Database.instance = new Database();
        }
        return Database.instance;
      }

      public query(sql: string): void {
        console.log(`Bajarilmoqda: ${sql}`);
      }
    }
    ```
  </TabsContent>

  <TabsContent value="java">
    ```java
    // Double-Checked Locking
    public class Database {
        private static volatile Database instance;

        private Database() {
            System.out.println("Ma'lumotlar bazasi instansiyasi yaratildi");
        }

        public static Database getInstance() {
            if (instance == null) {
                synchronized (Database.class) {
                    if (instance == null) {
                        instance = new Database();
                    }
                }
            }
            return instance;
        }
    }
    ```
  </TabsContent>

## Real misol: Logger

```typescript
class Logger {
  private static instance: Logger;
  private logs: string[] = [];

  private constructor() {}

  public static getInstance(): Logger {
    if (!Logger.instance) {
      Logger.instance = new Logger();
    }
    return Logger.instance;
  }

  public log(message: string): void {
    const timestamp = new Date().toISOString();
    const logEntry = `[${timestamp}] ${message}`;
    this.logs.push(logEntry);
    console.log(logEntry);
  }

  public getLogs(): string[] {
    return [...this.logs];
  }
}

// Foydalanish
const logger1 = Logger.getInstance();
const logger2 = Logger.getInstance();

logger1.log('Ilova ishga tushdi');
logger2.log('Foydalanuvchi tizimga kirdi');

console.log(logger1 === logger2); // true
console.log(logger1.getLogs());
```

## Afzalliklari

- ✅ Yagona, umumiy resursga nazorat qilinadigan kirish
- ✅ Kechiktirilgan initsializatsiya (lazy initialization) mumkin
- ✅ To'g'ri amalga oshirilganda thread-safe bo'ladi
- ✅ Amalga oshirish oson

## Kamchiliklari

- ❌ Yagona mas'uliyat tamoyilini (Single Responsibility Principle) buzadi
- ❌ Unit test qilishni qiyinlashtiradi (qattiq bog'liqlik)
- ❌ Bog'liqliklarni yashirishi mumkin
- ❌ Muammoni hal qilishdan ko'ra yomon dizaynni yashirishi mumkin

<Callout>
Singleton patterni ko'pincha anti-pattern deb hisoblanadi. Test qilish qulayligi va moslashuvchanlik uchun uning o'rniga **Dependency Injection**-dan foydalanishni ko'rib chiqing.
</Callout>

## Qachon ishlatish kerak

## Qachon ishlatmaslik kerak

- Bir nechta konfiguratsiyalar kerak bo'lganda
- Ko'p oqimli (multithreaded) muhitlarda (thread-safe muqobillardan foydalaning)
- Unit test qilish muhim bo'lganda
- Klassni kengaytirmoqchi (extend) bo'lganingizda

## Bog'liq patternlar

- **Dependency Injection**: Global kirish uchun yaxshiroq muqobil
- **Factory Method**: Singleton bilan birga ishlatilishi mumkin
- **Abstract Factory**: Singleton bilan birga ishlashi mumkin
